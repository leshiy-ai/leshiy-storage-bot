# Release Notes v3.0.0 - Миграция на Yandex.Cloud

Это мажорное обновление знаменует собой полный перенос инфраструктуры бота с Cloudflare Workers на платформу Yandex.Cloud. Изменения коснулись архитектуры, базы данных, способа конфигурации и процесса развертывания.

## Ключевые изменения

### 1. **Смена платформы**
- **С Cloudflare Workers на Yandex.Cloud Functions:** Бот больше не работает в пограничной среде (edge) V8 isolates. Теперь он функционирует как стандартная серверная Node.js-функция в облаке Яндекса. Это меняет подход к обработке запросов и управлению состоянием.

### 2. **Новый стек базы данных**
- **С Cloudflare D1/KV на Yandex Database (YDB):** Хранение данных полностью переработано.
  - Вместо SQL-совместимой D1 и key-value хранилища KV используется **Yandex Database** в режиме **Document Table** (NoSQL).
  - Для взаимодействия с базой данных написан новый модуль `db_adapter.js`, который инкапсулирует всю логику работы с YDB SDK.

### 3. **Архитектурные и кодовые изменения**
- **Точка входа:** Вместо `export default { fetch }` (стандарт Cloudflare) теперь используется `module.exports.handler` — стандартный обработчик для Yandex.Cloud Functions.
- **Конфигурация:** Убран специфичный для Cloudflare файл `wrangler.toml`. Вся конфигурация проекта (токены, ключи API, эндпоинты) теперь задается через **переменные окружения** непосредственно в настройках облачной функции.
- **Безопасность:** Все секреты и пароли удалены из кода. Реализована строгая привязка к переменным окружения (`process.env.VARIABLE_NAME`), что является более безопасной практикой.

### 4. **Новый процесс развертывания**
- **API Gateway:** Для публикации функции в интернете и получения публичного URL теперь используется **Yandex API Gateway**, который проксирует запросы к облачной функции.
- **Сборка проекта:** Деплой осуществляется путем загрузки ZIP-архива, содержащего весь код проекта, включая `node_modules`.

## Что это значит для разработчиков

- **Единая среда Node.js:** Разработка и отладка становятся более традиционными для Node.js-разработчиков.
- **Централизованное управление:** Вся инфраструктура (функция, база данных, API-шлюз) теперь управляется из консоли Yandex.Cloud.
- **Новый адаптер БД:** При работе с данными необходимо использовать функции из `db_adapter.js`, который обращается к YDB. Прямые SQL-запросы больше не работают.
