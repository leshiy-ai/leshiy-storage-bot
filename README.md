# 🗄 Telegram & VK Storage-bot "Хранилка" v2.4.0

Универсальный комбайн для облачного хранения и нейросетевой обработки данных.

Это не просто бот для хранения файлов, а ваш личный медиа-архивариус с искусственным интеллектом, который теперь работает и во ВКонтакте. Загружайте фото, видео, голосовые сообщения и документы в облачные хранилища (Яндекс.Диск, Google Drive, Dropbox, WebDAV). Мощные AI-функции проанализируют содержимое, создадут подробные описания и помогут вам легко находить и даже выгружать нужные файлы.

Бот работает на быстрой и надежной платформе Cloudflare Workers.
## 🚀 Ключевые возможности

    🚀 Прокачанная интеграция с VK:

        VK Mini App: Бот теперь работает как полноценное веб-приложение внутри ВКонтакте.

        Интерактивность: Поддержка инлайн-кнопок и сложных диалоговых сценариев.

        Ультимативный парсинг: Обход заглушек VK и прямой захват видео/документов даже из закрытых вложений.

        AI-анализ для VK: Фоновое описание контента теперь доступно и для файлов из ВКонтакте.

    🧠 Продвинутый AI-анализ (Gemini 2.5 Flash-lite):

        📄 Анализ документов: "Чтение" PDF/DOCX и создание кратких резюме.

        🎬 Глубокий анализ видео: Покадровый разбор действий, объектов и распознавание речи.

        👁️ Автоматическое описание фото: Создание детальных текстовых описаний для визуального поиска.

        🗣️ Распознавание речи: Мгновенное преобразование голосовых сообщений в текст.

    ⚙️ Гибкая настройка AI (/ai_settings): Управление моделями (Gemini, BotHub, Workers AI) на лету через интерактивное админ-меню.

    ☁️ Мульти-облачность: Единый интерфейс для Yandex, Google, Dropbox и WebDAV (Mail.ru).

    📂 Умное управление: Семантический поиск ("найди фото с собакой") и прямая выгрузка файлов из облака обратно в чат.

## 📖 Список команд
| Команда | Описание |
| :--- | :--- |
| `/start` | Приветствие, статус подключенных дисков и основное меню. |
| `/search`| Поиск файлов: обычный (по имени) или AI-поиск (по смыслу). |
| `/folder`| Выбор или создание папки для загрузки. |
| `/share` | Создание пригласительной ссылки с доступом к конкретной папке. |
| `/disconnect`| Отключение текущего хранилища или выход из общего доступа. |
| `/debug` | Технический аудит: ID, версия кода, статус базы данных. |
| `/admin` | (Admin Only) Статистика пользователей и лог ошибок. |
| `/ai_settings`| (Admin Only) Выбор провайдеров и моделей ИИ. |

## 🛠 Технологический стек

    Runtime: Cloudflare Workers (V8 Engine)

    Databases:

        Cloudflare D1 (SQL): Индексация файлов, хранение AI-описаний и тегов.

        Cloudflare KV: Сессии, OAuth-токены и конфигурации.

    AI Engine:

        Google Gemini: Основной мозг для анализа мультимедиа.

        Cloudflare Workers AI: Локальные модели (Whisper, Llama).

        BotHub: Резервный провайдер.

## ⚙️ Полная настройка и развертывание
### Шаг 1: Подготовка баз данных

    KV Namespace: Создайте KV с именем USER_DB.

    D1 Database: Создайте базу FILES_DB и выполните в консоли:

```sql
CREATE TABLE files (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    userId TEXT NOT NULL,
    fileName TEXT NOT NULL,
    fileId TEXT,
    fileType TEXT,
    provider TEXT,
    remotePath TEXT,
    timestamp INTEGER NOT NULL,
    ai_description TEXT,
    tags TEXT
);
```

### Шаг 2: Переменные окружения (Secrets)

Добавьте основные ключи в Settings > Variables:
| Ключ | Описание | Обязательно |
| :--- | :--- | :--- |
| `TELEGRAM_TOKEN` | Токен от @BotFather. | Да |
| `ADMIN_ID` | Ваш ID в Telegram. | Да |
| `GEMINI_API_KEY` | Ключ от Google AI Studio. | Да (для AI) |
| `YANDEX_CLIENT_ID`| OAuth ID для Яндекс.Диска. | Нет |
| `GOOGLE_CLIENT_ID`| OAuth ID из Google Console. | Нет |
| `VK_GROUP_TOKEN` | Токен сообщества для работы в VK. | Нет |

### Шаг 3: Настройка OAuth Callback

Для корректной авторизации укажите в настройках приложений провайдеров:
`https://<ваш-воркер>.<аккаунт>.workers.dev/auth/<yandex|google|dropbox|mailru>/callback`

### Шаг 4: Деплой

    Склонируйте репозиторий.

    Проверьте wrangler.toml на наличие привязок к D1 и KV.

    Выполните:

```bash
wrangler deploy
```

    Установите вебхук через браузер:
    `https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://ваша-ссылка.workers.dev`
