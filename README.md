# 🗄️ Storage Bot "Хранилка" by Leshiy

Это многофункциональный бот и Mini App, который работает в **Telegram**, **ВКонтакте** и **Одноклассниках**. Он служит «мостом» между вашими социальными сетями и облачными хранилищами, позволяя легко и бесплатно сохранять медиафайлы.

## 🚀 Основные возможности

*   **Мультиплатформенность:** Полноценная работа в Telegram (бот), ВКонтакте (бот + Mini App) и Одноклассниках (Mini App).
*   **Широкая интеграция с облаками:** Поддержка Яндекс.Диска, Google Drive, Dropbox, а также **любых** FTP, SFTP и WebDAV-совместимых хранилищ (включая Облако Mail.Ru).
*   **Умная настройка серверов:** В Mini App достаточно вставить одну ссылку-приглашение (например, `sftp://user:pass@host:port/path`), и приложение само разберет и заполнит все поля для подключения.
*   **Гибкая система доступа:**
    *   Делитесь доступом к своему хранилищу с друзьями через простые реферальные ссылки.
    *   Получайте **мгновенные уведомления** в Mini App, когда кто-то подключается к вашему облаку.
*   **Продвинутая админ-панель:** В Telegram-боте доступна мощная панель администратора с пагинацией для удобного управления пользователями и ссылками-приглашениями.
*   **Интеграция с ИИ:** Встроенный чат-ассистент на базе **Gemini** и **YandexGPT** поможет с любыми вопросами и просто поддержит беседу.
*   **Безопасность:** Для публичных сервисов используется авторизация через OAuth 2.0, что гарантирует безопасность ваших данных.

## 📦 Развертывание

Проект адаптирован для работы на платформе **Yandex.Cloud**.

### 1. Переменные окружения

Перед развертыванием необходимо настроить следующие переменные окружения в вашей Yandex.Cloud Function:

**Обязательные для всех:**
- `YDB_ENDPOINT`: Эндпоинт вашей Yandex Database.
- `YDB_DATABASE`: Путь к вашей базе данных.
- `TELEGRAM_TOKEN`: Токен вашего Telegram-бота.
- `VK_GROUP_TOKEN`: Токен доступа вашего сообщества VK.
- `VK_APP_ID`: ID вашего VK Mini App.
- `OK_APP_ID`: ID вашего OK Mini App.
- `VK_GROUP_ID`: ID вашего сообщества VK.
- `APP_DOMAIN`: Публичный домен вашего API Gateway.

**Для облачных хранилищ (OAuth):**
- `YANDEX_CLIENT_ID` & `YANDEX_CLIENT_SECRET`
- `GOOGLE_CLIENT_ID` & `GOOGLE_CLIENT_SECRET`
- `DROPBOX_CLIENT_ID` & `DROPBOX_CLIENT_SECRET`

**Для AI-сервисов:**
- `GEMINI_API_KEY`: API-ключ от Google AI Studio.
- `GEMINI_PROXY_KEY`: Секретный ключ для вашего прокси-сервера Gemini (если используется).
- `YANDEX_API_KEY`: API-ключ сервисного аккаунта Yandex.Cloud.
- `YANDEX_FOLDER_ID`: Идентификатор каталога в Yandex.Cloud.

### 2. Процесс развертывания

1.  **Создайте Yandex Database (YDB):** В консоли Yandex.Cloud создайте новую бессерверную базу данных.
2.  **Создайте сервисный аккаунт:** Сгенерируйте необходимые API-ключи и авторизованные ключи с ролями `ydb.editor` и `ai.editor`.
3.  **Создайте Yandex.Cloud Function:**
    *   Создайте новую функцию на Node.js.
    *   Укажите созданный сервисный аккаунт и все переменные окружения.
    *   Загрузите ZIP-архив с кодом проекта (включая `node_modules`).
    *   Точка входа: `index.js`, обработчик: `handler`.
4.  **Создайте API Gateway:**
    *   Настройте интеграцию шлюза с вашей функцией для всех запросов (`/*`).
    *   Опубликуйте шлюз для получения публичного URL.
5.  **Настройте вебхуки:**
    *   Укажите полученный URL в настройках вебхуков для Telegram-бота и Callback API для сообщества ВКонтакте.
