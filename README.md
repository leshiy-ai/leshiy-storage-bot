# 🗄 Storage Bot "Хранилка" by Leshiy

Это многофункциональный бот, который работает одновременно как Telegram-бот, VK-чат-бот и VK Mini App. Он служит «мостом» между вашими социальными сетями и облачными хранилищами, позволяя легко сохранять медиафайлы.

## 🚀 Основные возможности

- **Мультиплатформенность:** Работает в Telegram и ВКонтакте (чат-бот + Mini App).
- **Облачная интеграция:** Поддержка Яндекс.Диска, Google Drive, Dropbox, любых WebDAV-совместимых хранилищ (включая Облако Mail.Ru), а также FTP и SFTP серверов.
- **Безопасность:** Авторизация через OAuth 2.0, без хранения ваших паролей от облачных сервисов.
- **Гибкий доступ:** Возможность делиться доступом к своему хранилищу с друзьями и близкими через реферальные ссылки.
- **Умное именование:** Автоматическое и осмысленное именование файлов.
- **Интеграция с ИИ:** Встроенный чат-ассистент с поддержкой моделей **Gemini**, **YandexGPT** и других.

## 📦 Развертывание

1.  **Склонируйте репозиторий.**
2.  **Настройте переменные окружения:** Создайте и настройте облачную функцию в Yandex.Cloud, указав все необходимые переменные окружения (токены ботов, API-ключи, эндпоинты базы данных).
3.  **Установите зависимости:** `npm install`
4.  **Создайте ZIP-архив:** Упакуйте проект в ZIP-архив.
5.  **Загрузите архив:** Загрузите архив в вашу облачную функцию.
6.  **Настройте API Gateway:** Создайте и настройте API Gateway для получения публичного URL и привяжите его к функции.
7.  **Настройте вебхуки:** Укажите полученный URL в настройках ваших ботов (Telegram и VK) для получения обновлений.

Данная версия полностью адаптирована для работы на платформе **Yandex.Cloud**.

## Технологический стек

- **Платформа:** Yandex.Cloud Functions
- **Среда выполнения:** Node.js
- **База данных:** Yandex Database (YDB) в режиме Document Table (NoSQL)
- **AI:** YandexGPT, Gemini, Yandex SpeechKit
- **API Шлюз:** Yandex API Gateway
- **Зависимости:** `aws-sdk`, `yandex-cloud/nodejs-sdk`, `basic-ftp`, `ssh2-sftp-client`

## Ключевые отличия от версии для Cloudflare

- **Инфраструктура:** Проект переехал с глобальной пограничной сети Cloudflare на централизованную облачную платформу Яндекса.
- **База данных:** Вместо Cloudflare D1/KV используется Yandex Database. Это потребовало полной переработки слоя доступа к данным, который теперь вынесен в модуль `db_adapter.js`.
- **Конфигурация:** Настройка проекта осуществляется через переменные окружения в консоли Yandex.Cloud, а не через файл `wrangler.toml`.

## Настройка и развертывание

### 1. Переменные окружения

Перед развертыванием необходимо настроить следующие переменные окружения в вашей Yandex.Cloud Function. Это самый важный шаг для обеспечения безопасности и работоспособности бота.

**Обязательные для всех:**
- `YDB_ENDPOINT`: Эндпоинт вашей Yandex Database (например, `ydb.serverless.yandexcloud.net:2135`).
- `YDB_DATABASE`: Путь к вашей базе данных (например, `/ru-central1/b1g.../etn...`).
- `TELEGRAM_TOKEN`: Токен вашего Telegram-бота.
- `VK_GROUP_TOKEN`: Токен доступа вашего сообщества VK.
- `VK_APP_ID`: ID вашего VK Mini App.
- `VK_GROUP_ID`: ID вашего сообщества VK.
- `APP_DOMAIN`: Публичный домен вашего API Gateway.

**Для облачных хранилищ (OAuth):**
- `YANDEX_CLIENT_ID`: ID OAuth-приложения Яндекс.Диска.
- `YANDEX_CLIENT_SECRET`: Секрет OAuth-приложения Яндекс.Диска.
- `GOOGLE_CLIENT_ID`: ID клиента Google Drive API.
- `GOOGLE_CLIENT_SECRET`: Секрет клиента Google Drive API.
- `DROPBOX_CLIENT_ID`: Ключ приложения Dropbox.
- `DROPBOX_CLIENT_SECRET`: Секрет приложения Dropbox.

**Для AI-сервисов:**
- `GEMINI_API_KEY`: Ваш API-ключ от Google AI Studio.
- `YANDEX_API_KEY`: API-ключ сервисного аккаунта Yandex.Cloud.
- `YANDEX_FOLDER_ID`: Идентификатор каталога в Yandex.Cloud.

**Для FTP/SFTP:**
- Для этих протоколов не нужны отдельные переменные окружения. Данные для подключения (хост, порт, логин, пароль) передаются пользователем в чате.

### 2. Подготовка к развертыванию

1. **Установите зависимости:**
   ```bash
   npm install
   ```

2. **Создайте ZIP-архив:**
   Запакуйте все файлы проекта, включая папку `node_modules`, в один ZIP-архив. **Важно:** `index.js` должен находиться в корне архива.

### 3. Процесс развертывания

1. **Создайте Yandex Database (YDB):** В консоли Yandex.Cloud создайте новую бессерверную базу данных.
2. **Создайте сервисный аккаунт:** Создайте сервисный аккаунт с ролью `ydb.editor` и `ai.editor`, сгенерируйте для него авторизованные ключи и API-ключ.
3. **Создайте Yandex.Cloud Function:**
   - Создайте новую функцию на Node.js.
   - В настройках функции укажите созданный сервисный аккаунт.
   - В разделе "Переменные окружения" добавьте все ключи и токены, описанные в шаге 1.
   - Загрузите созданный ZIP-архив с кодом.
   - В качестве точки входа укажите `index.js`, а в качестве обработчика — `handler`.
4. **Создайте API Gateway:**
   - Создайте новый API-шлюз.
   - В спецификации шлюза настройте интеграцию с вашей функцией, которая будет вызываться для всех запросов (`/*`).
   - Опубликуйте шлюз, и вы получите публичный URL для вашего бота.

5. **Настройте вебхуки:**
   - **Telegram:** Установите вебхук для вашего Telegram-бота, указав URL вашего API Gateway.
   - **VK:** В настройках вашего сообщества VK укажите тот же URL в качестве Callback API.

    Установите вебхук через браузер:
    `https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://<domain>/`
