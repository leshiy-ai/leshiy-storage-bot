# 🤖 Умный Telegram-бот "Хранилка" v2.3.0 с AI-анализом видео и документов

Это не просто бот для хранения файлов, а ваш личный медиа-архивариус с искусственным интеллектом. Загружайте фото, видео, голосовые сообщения и **документы** в облачные хранилища (Яндекс.Диск, Google Drive, Dropbox, WebDAV). Мощные AI-функции **проанализируют содержимое**, создадут подробные описания и помогут вам легко находить и даже **выгружать** нужные файлы.

Бот работает на быстрой и надежной платформе **Cloudflare Workers**.

## 🚀 Ключевые возможности

*   **🧠 Продвинутый AI-анализ:**
    *   **📄 Анализ документов:** Бот "читает" и анализирует содержимое ваших документов (PDF, DOCX и др.), создавая краткое резюме для быстрого поиска.
    *   **🎬 Глубокий анализ видео:** Новая AI-модель предоставляет полный покадровый разбор видео, описывая действия, объекты и распознавая речь.
    *   **👁️ Автоматическое описание фото:** Бот "смотрит" на ваши изображения и создает для них детальные текстовые описания.
    *   **🗣️ Распознавание речи:** Отправляйте голосовые сообщения, и бот автоматически преобразует их в текст.
*   **⚡️ Оптимизированные AI-модели:** Большинство AI-задач теперь выполняются на быстрой и экономичной модели `gemini-2.5-flash-lite`, что обеспечивает мгновенный отклик.
*   **⚙️ Гибкая настройка AI (`/admin`, `/ai_settings`):** Выбирайте, какие AI-модели использовать для разных задач (чат, анализ изображений, распознавание речи, анализ видео и документов) через удобное меню в админ-панели.
*   **☁️ Мульти-облачная интеграция:** Поддержка Яндекс.Диска, Google Drive, Dropbox и любых WebDAV-совместимых хранилищ (включая Облако Mail.ru).
*   **📂 Управление файлами:**
    *   **⚡️ Умный поиск (`/search`):** Мощная система поиска, которая автоматически выбирает нужный режим. Для простых запросов (имя файла, тег) используется мгновенный поиск по базе, а для сложных, "человеческих" запросов ("найди фото с собакой на пляже") — включается оптимизированный AI-поиск.
    *   **Выгрузка из облака:** Нашли нужный файл? Теперь его можно скачать прямо в Telegram!
*   **🤝 Улучшенная система приглашений (`/share`):** Делитесь доступом к своему облаку с друзьями, привязав ссылку к **конкретной папке**.

## 📖 Список команд

| Команда | Описание |
| :--- | :--- |
| `/start` | Показать приветственное сообщение, статус подключения и основное меню. |
| `/search` | Начать поиск. Поддерживает как простой поиск по имени/тегам, так и сложный семантический поиск по описанию. |
| `/folder` | Выбрать папку для загрузки файлов или создать новую. |
| `/share` | Создать пригласительную ссылку для друга, привязанную к текущей папке. |
| `/disconnect` | Отключить текущий диск или отсоединиться от облака друга. |
| `/debug` | Показать техническую информацию (ID, версия, текущая папка). |
| `/admin` или `/ai_settings` | **(Админ)** Показать панель со списком пользователей и кнопкой настроек AI. |

## 🛠 Технологический стек

*   **Платформа**: Cloudflare Workers
*   **Язык**: JavaScript
*   **Базы данных**:
    *   **Cloudflare D1**: Для хранения метаданных файлов, их AI-описаний и тегов.
    *   **Cloudflare KV**: Для хранения токенов и пользовательских настроек.
*   **AI**:
    *   **Google Gemini**: Для анализа изображений, видео, документов (модель `gemini-2.5-flash-lite`), распознавания речи и ведения диалога.
    *   **Cloudflare Workers AI**: Для встроенных моделей (включая Whisper).
    *   **BotHub**: Как альтернативный провайдер для различных AI-моделей.
*   **Хранилища**: Яндекс.Диск, Google Drive, Dropbox, WebDAV.

## ⚙️ Полная настройка и развертывание

### Шаг 1: Настройка баз данных

Вам понадобится два типа хранилищ Cloudflare.

#### 1. KV Namespace (для настроек пользователей)

1.  Создайте KV Namespace в панели Cloudflare (`Dashboards` > `Workers & Pages` > `KV`).
2.  Привяжите его к вашему воркеру (в настройках воркера > `Settings` > `Bindings`).
3.  **Имя переменной привязки (Variable name)** должно быть `USER_DB`.

#### 2. D1 Database (для AI-описаний и файлов)

1.  Создайте базу данных D1 (`Dashboards` > `Workers & Pages` > `D1`).
2.  Привяжите ее к воркеру с **именем переменной `FILES_DB`**.
3.  После создания базы перейдите в ее консоль и выполните следующий SQL-запрос для создания таблицы:

```sql
CREATE TABLE files (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    userId TEXT NOT NULL,
    fileName TEXT NOT NULL,
    fileId TEXT,
    fileType TEXT,
    provider TEXT,
    remotePath TEXT,
    timestamp INTEGER NOT NULL,
    ai_description TEXT,
    tags TEXT
);
```

### Шаг 2: Настройка переменных окружения (Секретов)

Перейдите в настройки вашего воркера (`Settings` > `Variables`) и добавьте следующие секреты (`Encrypt`):

| Ключ | Описание | Обязательно |
| :--- | :--- | :--- |
| `TELEGRAM_TOKEN` | Токен вашего Telegram-бота от [@BotFather](https://t.me/BotFather). | **Да** |
| `ADMIN_ID` | Ваш ID в Telegram. | **Да** |
| `BOT_USERNAME`| Имя пользователя вашего бота (например, `my_super_bot`). | **Да** |
| `GEMINI_API_KEY` | API-ключ от Google AI Studio для Gemini. | **Рекомендуется** |
| `BOTHUB_API_KEY` | API-ключ от BotHub для доступа к альтернативным моделям. | Нет |
| `YANDEX_CLIENT_ID` | ID приложения [Яндекс.OAuth](https://oauth.yandex.ru/). | Нет |
| `YANDEX_CLIENT_SECRET` | Секрет приложения Яндекс.OAuth. | Нет |
| `GOOGLE_CLIENT_ID` | Client ID из [Google Cloud Console](https://console.cloud.google.com/apis/credentials). | Нет |
| `GOOGLE_CLIENT_SECRET` | Client Secret из Google Cloud Console. | Нет |
| `DROPBOX_CLIENT_ID` | App key из [Dropbox App Console](https://www.dropbox.com/developers/apps). | Нет |
| `DROPBOX_CLIENT_SECRET`| App secret из Dropbox App Console. | Нет |

> **Примечание:** Бот будет работать и без ключей облачных провайдеров, но пользователи не смогут к ним подключиться. AI-функции требуют наличия хотя бы одного из ключей (`GEMINI_API_KEY`, `BOTHUB_API_KEY` или настроенного Workers AI).

### Шаг 3: Настройка OAuth (Callback URLs)

При регистрации приложений у провайдеров (Яндекс, Google, Dropbox) укажите следующие URL для перенаправления:

-   **Яндекс**: `https://<ваш-воркер>.<ваш-аккаунт>.workers.dev/auth/yandex/callback`
-   **Google**: `https://<ваш-воркер>.<ваш-аккаунт>.workers.dev/auth/google/callback`
-   **Dropbox**: `https://<ваш-воркер>.<ваш-аккаунт>.workers.dev/auth/dropbox/callback`

### Шаг 4: Развертывание

1.  Установите [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/get-started/).
2.  Авторизуйтесь в Cloudflare: `wrangler login`.
3.  Поместите код из `worker.js` в ваш проект.
4.  Выполните команду `wrangler deploy`.
5.  Убедитесь, что все переменные и привязки баз данных настроены в панели Cloudflare.
6.  Установите вебхук для Telegram-бота, выполнив в браузере запрос:<br>
    `https://api.telegram.org/bot<TELEGRAM_TOKEN>/setWebhook?url=https://<ваш-воркер>.<ваш-аккаунт>.workers.dev`
