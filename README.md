# 🤖 Telegram Storage Bot "Хранилка" v2.1.3

Личный Telegram-бот для автоматической загрузки фото, видео и документов в облачные хранилища Яндекс.Диск, Google Drive, Dropbox и Mail.ru.

Новая версия бота работает на быстрой и надежной платформе **Cloudflare Workers**, обеспечивая мгновенную обработку файлов и команд.

## ✨ Основные функции

*   **Облачная интеграция**: Прямое подключение к **Яндекс.Диску**, **Google Drive**, **Dropbox** и **Облаку Mail.ru** (через WebDAV).
*   **Система приглашений**: Новая команда `/share` позволяет создать уникальную ссылку, чтобы поделиться доступом к своему облаку с другом без передачи логинов и паролей.
*   **Управление папками**: Команда `/folder` позволяет выбрать существующую папку для загрузки или создать новую прямо из бота.
*   **Простое подключение**: Пользователи авторизуются один раз через команду `/start` и могут отправлять файлы.
*   **Гибкое администрирование**:
    *   Команда `/admin` для просмотра списка всех пользователей.
    *   Команда `/add [ID]` для принудительного добавления пользователя и настройки папки для него.
*   **Диагностика**: Команда `/debug` для проверки статуса бота, ID пользователя и текущей папки.
*   **Веб-статус**: Простая веб-страница для проверки работоспособности сервиса.

## 📖 Список команд

| Команда       | Описание                                                                                |
| :------------ | :-------------------------------------------------------------------------------------- |
| `/start`      | Показать приветственное сообщение, статус подключения и основное меню.                  |
| `/folder`     | Выбрать папку для загрузки файлов или создать новую.                                   |
| `/share`      | **(Новое!)** Создать пригласительную ссылку для друга, чтобы он мог использовать ваше облако. |
| `/disconnect` | Отключить текущий диск или отсоединиться от облака друга.                               |
| `/debug`      | Показать техническую информацию (ID, версия, текущая папка).                           |
| `/admin`      | **(Только для админа)** Показать панель администратора со списком пользователей.          |
| `/add [ID]`   | **(Только для админа)** Предоставить пользователю доступ к вашему облаку.                 |


## 🛠 Технологический стек

*   **Платформа**: Cloudflare Workers
*   **Язык**: JavaScript
*   **База данных**: Cloudflare KV (для хранения токенов и данных пользователей)
*   **Хранилища**: Яндекс.Диск, Google Drive, Dropbox, Mail.ru (WebDAV)

## ⚙️ Переменные окружения и настройка

Для работы бота необходимо зарегистрировать приложения в облачных сервисах, а затем настроить переменные (Secrets) в вашем Cloudflare Worker.

| Ключ                   | Описание                                                                                                |
| :--------------------- | :------------------------------------------------------------------------------------------------------ |
| `TELEGRAM_TOKEN`       | Токен вашего Telegram-бота, полученный от [@BotFather](https://t.me/BotFather).                         |
| `ADMIN_ID`             | ID администратора бота в Telegram.                                                                      |
| `BOT_USERNAME`         | Имя пользователя вашего бота (например, `my_super_bot`) для генерации корректных ссылок.              |
| `YANDEX_CLIENT_ID`     | ID вашего приложения в [Яндекс.OAuth](https://oauth.yandex.ru/).                                         |
| `YANDEX_CLIENT_SECRET` | Секретный ключ вашего приложения в Яндекс.OAuth.                                                        |
| `GOOGLE_CLIENT_ID`     | Client ID вашего приложения в [Google Cloud Console](https://console.cloud.google.com/apis/credentials). |
| `GOOGLE_CLIENT_SECRET` | Client Secret вашего приложения в Google Cloud Console.                                                 |
| `DROPBOX_CLIENT_ID`    | App key вашего приложения в [Dropbox App Console](https://www.dropbox.com/developers/apps).              |
| `DROPBOX_CLIENT_SECRET`| App secret вашего приложения в Dropbox App Console.                                                     |
| `MAILRU_CLIENT_ID`     | ID вашего приложения в [Кабинете разработчика VK](https://dev.vk.com/ru/apps). (для WebDAV не нужен) |
| `MAILRU_CLIENT_SECRET` | Секретный ключ вашего приложения. (для WebDAV не нужен)                                                  |

**Callback URL для OAuth приложений:**

*   **Яндекс**: `https://<ваш-воркер>.<ваш-аккаунт>.workers.dev/auth/yandex/callback`
*   **Google**: `https://<ваш-воркер>.<ваш-аккаунт>.workers.dev/auth/google/callback`
*   **Dropbox**: `https://<ваш-воркер>.<ваш-аккаунт>.workers.dev/auth/dropbox/callback`

**KV Namespace:**

Необходимо создать KV Namespace и привязать его к воркеру с именем `USER_DB`.

## 🚀 Развертывание

1.  Установите [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/get-started/).
2.  Авторизуйтесь в своей учетной записи Cloudflare (`wrangler login`).
3.  Настройте `wrangler.toml` (если используете).
4.  Выполните команду `wrangler deploy`.
5.  Настройте переменные окружения в панели управления Cloudflare.
6.  Установите вебхук для Telegram-бота:
    `https://api.telegram.org/bot<TELEGRAM_TOKEN>/setWebhook?url=https://<ваш-воркер>.<ваш-аккаунт>.workers.dev`
